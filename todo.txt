## Problem Analysis
The babel plugin in the loader causes **recursive load hook calls** when transforming files:
1. Load hook calls babel transform on `app/app.ts`
2. Babel loads its plugins (e.g., `babel-plugin-ember-template-compilation`)
3. Loading babel plugins triggers the load hook again
4. This creates recursion

## Possible Root Causes
1. **Loader not initializing**: The loader.mjs might have a syntax error or initialization issue
2. **Babel hanging silently**: The isTransforming flag might prevent ALL transforms
3. **Load hook blocking**: The early return logic might be too aggressive
4. **Missing dependencies**: Some babel plugin might not be loading correctly

### 3. Alternative Solutions

#### Option A: Use esbuild Instead of Babel
```typescript
import { transformSync } from 'esbuild';

export function createTypeScriptPlugin() {
  return {
    name: 'typescript',
    transform(code: string, id: string) {
      if (!id.endsWith('.ts') && !id.endsWith('.tsx')) {
        return null;
      }
      const result = transformSync(code, {
        loader: 'ts',
        format: 'esm',
        target: 'esnext',
      });
      return { code: result.code };
    },
  };
}
```

#### Option B: Pre-compile TypeScript
- Use tsc or esbuild to compile TS → JS before loader runs
- Loader only handles module resolution and HMR
- Cleaner separation of concerns

#### Option C: Isolate Babel in Worker Thread
- Run babel transform in a worker thread with its own context
- Prevents load hook recursion by isolating execution
- More complex but keeps babel functionality
